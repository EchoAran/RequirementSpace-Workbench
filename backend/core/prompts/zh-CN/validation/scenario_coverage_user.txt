{user_requirements}

## 目标组合
功能: {feature_name} ({feature_description})
参与者: {actor_name} ({actor_description})

## 项目中已有场景
{scenarios_json}

## 要求
分析每个已有场景的内容，判断是否可复用（feature 归属明显错误）：
1. 如果某个 scenario 明显应属于当前 feature（内容与当前 feature 完全匹配，当前 feature_id 明显是错误归属），输出 repair_draft 更新 feature_id。
2. 如果多个 scenario 都可能，输出 choice_group。
3. 如果无合适复用场景，设置 fallback_to_registry=true（走创建新场景流程）。
4. 只修改 feature_id，不改 actor_id。

输出 JSON 格式:
{{
  "candidates": [
    {{
      "repair_type": "reassign_scenario_feature",
      "title": "将场景「{scenario_name}」归属到当前功能",
      "rationale": "该场景内容与当前功能描述高度匹配",
      "confidence": 0.85,
      "patch": {{
        "updateNodes": [
          {{
            "kind": "scenario",
            "id": {scenario_id},
            "feature_id": {feature_id}
          }}
        ]
      }},
      "requires_user_decision": false
    }}
  ],
  "fallback": {{
    "kind": "generation_draft",
    "reason": "没有可复用的已有场景"
  }}
}}
